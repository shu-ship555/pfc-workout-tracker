---
name: refactor
description: コードベースの重複・デザイン違反を分析し、承認後にリファクタリングを実行する
---

プロジェクトルートにある `AGENTS.md` と `DESIGN.md` を読み込み、開発ルールを理解してください。

【タスク】
現在のコードベース（特に `src/` ディレクトリ配下）を分析し、ロジックやUIの重複がないか確認してください。

【要件】
1. DESIGN.md のデザインルールに則っているか確認し、修正が必要な箇所をすべてリストアップしてください。
2. DRY 原則に反している重複箇所やパターンをすべてリストアップしてください。
3. それらを共通コンポーネントやカスタムフックとしてどのように共通化するか、具体的な計画を提案してください。その際、`DESIGN.md` のスタイル規約に準拠させること。
4. 私が計画を承認した後にリファクタリングを実行し、`npm run lint` を走らせてコード品質に問題がないことを確認してください。
5. 1〜4 までの工程で README.md に修正の必要が出たら、修正してください。

まずは、見つかった重複箇所のリストアップから始めてください。
