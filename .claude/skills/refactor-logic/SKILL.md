---

プロジェクトルートにある `AGENTS.md` を読み込み、開発ルールを理解してください。

【タスク】
`src/` ディレクトリ配下のロジックを分析し、DRY・KISS・YAGNI 原則に反している箇所を洗い出してください。
UIスタイルの問題は対象外とし、ロジック・型・状態管理に絞って確認してください。

【要件】
1. 以下の観点で問題箇所をすべてリストアップしてください:
   - DRY: 複数ファイルに重複しているロジック・型定義・定数
   - カスタムフック: 同じ状態管理パターンが複数コンポーネントに散在している箇所
   - API クライアント: `api-client.ts` を使わずに直接 fetch を呼んでいる箇所
   - 型定義: `types.ts` に集約すべき型が個別ファイルに定義されている箇所
   - ユーティリティ: `lib/` に切り出せる汎用ロジックがコンポーネント内に書かれている箇所
2. 重複箇所ごとに「ファイル・内容・共通化の方針」を明記してください。
3. 私が計画を承認した後にリファクタリングを実行し、`npm run lint` でエラーがないことを確認してください。
4. README.md に修正が必要な場合は更新してください。

まずは重複箇所のリストアップから始めてください。
