## Design Enforcement
- UIの実装や修正を行う際は、必ずプロジェクトルートにある `DESIGN.md` を参照してください。
- `DESIGN.md` に定義されているカラーパレット、タイポグラフィ、スペーシングの規則を逸脱しないでください。
- 新しいコンポーネントを作成する場合は、`DESIGN.md` の設計思想に基づいたスタイルを提案してください。

## Code Quality & Refactoring
- コードの品質は、実装後に必ず `npm run lint` を実行し、エラーがないことを確認することで担保してください。
- DRY（Don't Repeat Yourself）の原則を徹底してください。
- ロジックやUIの重複を検知した場合は、作業前に以下の手順を踏んでください：
  1. 重複している箇所をリストアップし、ユーザーに報告する。
  2. それらをどのように共通化（共通コンポーネント化やカスタムフック化など）すべきか、具体的な計画を提示する。
  3. 提示した計画についてユーザーの承認を得た後、実装を実行する。
