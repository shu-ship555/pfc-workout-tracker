## タイポグラフィ

### フォント

| 用途               | フォント                                 |
| ------------------ | ---------------------------------------- |
| 本文・UI全般       | Montserrat（欧文）+ Noto Sans JP（和文） |
| 数値・コード・時刻 | Geist Mono（`font-mono`）                |

和欧混植構成。ブラウザはMontserratでラテン文字を描画し、グリフのない日本語はNoto Sans JPにフォールバックする。

### テキストサイズ

| 役割                     | クラス                             |
| ------------------------ | ---------------------------------- |
| ページタイトル           | `text-lg font-bold tracking-tight` |
| セクション見出し         | `text-sm font-medium`              |
| 本文・ラベル             | `text-sm`                          |
| 補助テキスト・メタデータ | `text-xs text-muted-foreground`    |
| データ値（PFCなど）      | `text-xl font-bold font-mono`      |
| 単位                     | `text-xs font-normal ml-0.5`       |

### ルール

- 数値は必ず `font-mono` を使う。重量・セット数・カロリーなどすべて統一する
- 補助テキストは `text-muted-foreground` で視覚的階層を作る
- `whitespace-nowrap` を使い、重要なラベルが改行されないよう保護する
- **最小フォントサイズは `text-[10px]`（10px相当）とする。それより小さいサイズは使わない**
- px換算で偶数に揃うようにする

## カラー

### パレット

Tailwindのハードコード色（`gray-500` など）は基本的に使わない。カラーは全て **`src/lib/color-constants.ts`** で一元管理する。コンポーネントに色文字列を直接書かず、必ず同ファイルの定数をインポートして使うこと。

| トークン                                 | 用途                     |
| ---------------------------------------- | ------------------------ |
| `bg-background` / `text-foreground`      | ページ背景・本文         |
| `bg-card` / `text-card-foreground`       | カード背景               |
| `bg-muted` / `text-muted-foreground`     | 非強調背景・補助テキスト |
| `bg-primary` / `text-primary-foreground` | メインアクション         |
| `border-border`                          | 区切り線・枠線           |
| `text-destructive`                       | 削除・危険操作           |

### セマンティックカラー（Tone on Tone ルール）

視覚的な統一感を持たせるため、**Tone on Tone（同系色の濃淡組み合わせ）** を原則とする。
背景色には不透明度10%（`/10`）を適用し、その上に同系統の濃い色のテキストを配置する。

| 意味           | 背景（10%透過）    | テキスト色（ライト / ダーク）             | チャート色            | 用途例                         |
| -------------- | ------------------ | ----------------------------------------- | --------------------- | ------------------------------ |
| 青（情報）     | `bg-blue-500/10`   | `text-blue-600 dark:text-blue-400`        | `hsl(221 83% 53%)`    | P タンパク質、睡眠チャート     |
| 黄（注意）     | `bg-yellow-500/10` | `text-yellow-700 dark:text-yellow-400`    | —                     | F 脂質、気分（中間）、反動あり |
| 緑（良好）     | `bg-green-500/10`  | `text-green-700 dark:text-green-400`      | `#16a34a`             | C 炭水化物、気分チャート       |
| 橙（高温）     | `bg-orange-500/10` | `text-orange-700 dark:text-orange-400`    | `#f97316`             | カロリー、最高気温チャート     |
| 赤（警告）     | `bg-red-500/10`    | `text-red-700 dark:text-red-400`          | —                     | 危険操作、静止不足、気分（悪） |
| シアン（低温） | —                  | —（テキスト用途では使用しない）           | `#06b6d4`             | 最低気温チャート               |

テキスト色は WCAG 2.1 AA（通常テキスト 4.5:1 以上）を満たす濃度に固定している。ライトモードでは `-700`、ダークモードでは `-400` を使う。`-500` を**テキスト色**として使うことはしない（白背景でコントラスト不足のため）。チャートのストロークや `bg-{color}-500/10` のような塗りつぶしで `-500` を使うのは可。

### チャートライン

| 役割           | 色                                       |
| -------------- | ---------------------------------------- |
| 重量推移ライン | `stroke-primary`（テーマ追従）           |
| 目標ライン     | `hsl(221 83% 53%)`（blue-600相当、破線） |

### ルール

- **カラーはTone on Toneで統一する**: 背景を薄く（`/10`）、文字を濃くすることで、コントラストを保ちつつモダンな印象にする。
- **セマンティックカラーの厳守**: 上記パレットに限定し、アドホックな中間色は使わない。
- **装飾目的の禁止**: 意味のない箇所に色を付けない。
- **ダークモード対応**: カラートークンおよび `dark:` プレフィックスを使い、ハードコードを避ける。

## スペーシング

### 基本単位

スペーシングの基本単位は `6`（1.5rem）。

数値（px・rem・Tailwindスケール）は **2・4・8の倍数** で揃える。

| 用途                   | 使う倍数の例           |
| ---------------------- | ---------------------- |
| padding / margin       | 2, 4, 8, 12, 16, 24 …  |
| gap                    | 2, 4, 8                |
| サイズ（width/height） | 4, 8, 16, 32, 40, 48 … |

中途半端な値（例: `p-3`・`gap-3`・`h-7`）は既存コンポーネントとの整合が取れている場合のみ許容し、新規追加では原則使わない。

| 場面                                | クラス           |
| ----------------------------------- | ---------------- |
| ページ内セクション間                | `space-y-6`      |
| セクション区切り線（Separator）周辺 | `my-12`          |
| カード内コンテンツ                  | `gap-3`〜`gap-4` |
| アイコンとテキスト                  | `gap-1`〜`gap-2` |
| ページ横余白                        | `px-4`           |
| ページ縦余白                        | `py-6`           |
| SP下部FABによるコンテンツ余白       | `pb-24 sm:pb-6`  |

### 上方錯視の補正

視覚的に上下の余白が等しく見えるよう、上の余白を下より小さくする。等量のpaddingを指定すると、人間の目には下側が広く見える（上方錯視）ため、上を約90%に抑えることで光学的な均衡を保つ。

```tsx
// 例: py-6 の代わりに
className="pt-5 pb-6"

// CardHeader: pb-2 に対して pt を1段小さく
className="pt-3 pb-2"
```

モーダル・カード・セクション内など、上下に余白が生じる箇所全般に適用する。

### レイアウト幅

```
max-w-5xl mx-auto w-full
```

コンテンツは最大幅 `max-w-5xl`（64rem）で中央寄せ。

## ボーダーラジウス

CSS変数 `--radius: 0.625rem` を基準とした倍率系。

| トークン       | 計算式        | 主な用途             |
| -------------- | ------------- | -------------------- |
| `rounded-sm`   | × 0.6         | バッジ内など小要素   |
| `rounded-md`   | × 0.8         | インプット、ボタン小 |
| `rounded-lg`   | × 1.0（基準） | カード、ボタン       |
| `rounded-xl`   | × 1.4         | モーダルなど         |
| `rounded-full` | —             | FABボタン            |

## コンポーネント

### Card

グループ化されたコンテンツの基本コンテナ。

```tsx
<Card>
  <CardHeader className="pb-2">...</CardHeader>
  <CardContent>...</CardContent>
</Card>
```

- カードの中にカードは入れない
- ヘッダーは `pb-2` で詰める

### Button

```tsx
// プライマリアクション
<Button className="hover:bg-primary/80">...</Button>

// 危険操作
<Button className="bg-destructive text-white hover:bg-destructive/90">...</Button>

// ゴーストボタン（アイコン操作）
<Button size="icon" variant="ghost" className="h-7 w-7">...</Button>
```

- プライマリボタンには必ず `hover:bg-primary/80` を付ける（デフォルトはアンカー内限定のため）
- 削除ボタンは `text-white`（`text-destructive-foreground` は未定義のため使用しない）

### Dialog vs AlertDialog

| 用途                   | コンポーネント |
| ---------------------- | -------------- |
| フォーム・メモ表示     | `Dialog`       |
| 削除確認など破壊的操作 | `AlertDialog`  |

### Table

- ヘッダー行は `<TableRow className="hover:bg-transparent">` でホバーなし
- データ行はデフォルト（ホバーあり）

### Badge

| 用途                            | variant     |
| ------------------------------- | ----------- |
| 部位タグ（胸・腕など）          | `secondary` |
| フラグ（WU・NEG・反動・不安定） | `outline`   |

フラグの色分け（`src/lib/color-constants.ts` の `FLAG_COLORS` を使用）：
- 反動あり：`text-yellow-700 dark:text-yellow-400`（`FLAG_COLORS.rebound`）
- 静止できていない：`text-red-700 dark:text-red-400`（`FLAG_COLORS.unstable`）

### Select

種目セレクトは長い選択肢が途切れないよう `SelectContent` に `min-w-max` を付ける。

```tsx
<SelectContent className="min-w-max">
```

## 日付・時刻

- すべての日付処理はJST（UTC+9）基準
- 日付ロジックを直接書かず、**`src/lib/date-utils.ts`** のユーティリティ関数を使う

```ts
import { jstToday, jstDaysAgo, formatDateTime, formatDateShort } from "@/lib/date-utils";

jstToday()           // 今日の YYYY-MM-DD
jstDaysAgo(3)        // 3日前の YYYY-MM-DD
formatDateTime(iso)  // MM/DD HH:MM（テーブル用）
formatDateShort(iso) // M月D日（チャートX軸用）
```

- チャートの表示フォーマットは `M月D日` 形式（`ja-JP` locale）
- テーブルの日時表示は `MM/DD HH:MM` 形式

## アクセシビリティ

- フォームの `<input>` には必ず `id` と対応する `<Label htmlFor>` を付ける
- 破壊的操作は必ず `AlertDialog` で確認を挟む
- ロード中は `<Skeleton>` でプレースホルダーを表示する
- 空状態は専用のメッセージUIを用意する（単純な非表示はしない）

### コントラスト比

WCAG 2.0 / 2.1 に準拠し、テキストと背景のコントラスト比を確保する。

| 基準                    | 通常テキスト | 大きなテキスト（18pt / 24px 以上、または 14pt / 18.66px 以上の太字） |
| ----------------------- | ------------ | -------------------------------------------------------------------- |
| レベル AA（最低基準）   | 4.5:1 以上   | 3:1 以上                                                             |
| レベル AAA（推奨基準）  | 7:1 以上     | 4.5:1 以上                                                           |

- 本プロジェクトは **レベル AA を最低基準** とする
- `text-muted-foreground` などの補助テキストも AA を満たすようカラートークンを調整する
- アイコンや UI コンポーネントの境界線も **3:1 以上** のコントラストを確保する
